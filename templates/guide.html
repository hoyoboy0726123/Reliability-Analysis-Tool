🔥 活化能 (Ea)
核心參數
定義：
失效機制啟動所需的最小能量障礙 (eV)
⚠️ 重要性：
Ea 是 Arrhenius 模型的核心，決定溫度加速效應的強度

元件/失效機制 Ea (eV) 說明
🔴 高溫極度敏感 (Ea ≥ 1.0)
IC 金-鋁鍵合線 1.0-1.26 金屬間化合物生長
塑膠材料老化 0.8-1.2 化學降解
金屬膜電阻 0.8-1.0 薄膜氧化漂移
🟡 半導體 IC 元件 (Ea = 0.6-1.0)
CPU/GPU 銅互連 ~0.9 電遷移 (EM)
MOS 氧化層 (TDDB) 0.6-0.9 內因性崩潰
鋁互連 0.55-0.6 舊技術 EM
🟢 業界默認值 (Ea = 0.7)
通用默認 0.7 JEDEC 標準，未知機制
碳成分電阻 0.7-0.9 阻值漂移
電容器 (AEC/TEC) 0.6-0.9 電解液蒸發
分立半導體 0.6-0.7 結點退化
🔵 機械/焊點 (Ea = 0.5-0.7)
焊點熱疲勞 0.5-0.7 溫度敏感度較低
焊點電遷移 0.6-0.8 高電流密度
⚫ 特殊機制
氧化層缺陷 (TDDB) 0.3 外因性，早期失效
熱載子注入 (HCI) -0.1 ~ -0.2 負Ea！低溫加速

Ea 代表什麼？

活化能是原子或分子系統從基態躍遷到能啟動特定失效過程所需的最小能量障礙。Ea 越高，該失效機制對溫度越敏感。

為什麼 Ea 如此重要？
  • 決定加速強度：Ea = 0.9 eV vs. 0.7 eV，溫度每升高 10°C，加速因子差異可達 30%
  • 不同機制有不同 Ea：一個元件可能有多個潛在失效模式，必須識別主導機制
  • 負 Ea 的挑戰：HCI 具負 Ea，低溫反而加速失效，顛覆常規認知
業界默認 0.7 eV 的局限性：

0.7 eV 是統計平均值，非物理精確值。對現代高性能 IC (CPU/GPU)，錯用 0.7 eV 會嚴重低估溫度影響。

⚠️ 關鍵原則：

準確的可靠度評估必須基於特定失效機制的 Ea，而非通用默認值。當無實驗數據時，參考同類元件文獻或材料供應商數據。

方法 1：Arrhenius 繪圖 (實驗法)

1. 選擇至少 3 個溫度 (如 85°C, 100°C, 125°C)
2. 每個溫度測試 20-30 樣品，記錄失效時間
3. 繪製 ln(MTTF) vs. 1/T (Kelvin)
4. 直線斜率 = -Ea/k，求出 Ea

方法 2：查詢文獻與標準

• JEDEC Publication 122C：提供多種失效機制 Ea 範圍
• 材料供應商數據表
• 同類產品可靠度報告
• 學術論文 (IEEE, IEDM)

方法 3：失效物理分析 (PoF)

通過失效分析 (FA) 確定主導失效機制：
• SEM/TEM：觀察微觀結構
• 電性分析：確定失效模式
• 匹配已知機制的 Ea

⚠️ 注意：

• 零失效測試：JEDEC 建議使用 Ea = 1.0 eV (保守)
• 多重機制：總失效率是所有模式之和
• 溫度範圍：Ea 在不同溫度範圍可能變化

🌡️ 溫度加速因子
Arrhenius
公式：
AFT = exp[(Ea/k) × (1/Tuse - 1/Talt)]

參數 說明 典型值
Tuse 使用溫度 (°C) 25-35°C
Talt 測試溫度 (°C) 60-125°C
Ea 活化能 (eV) 0.3-1.2 eV

  • • 電子元件：Ea = 0.7-1.0 eV
  • • 焊點：Ea = 0.5-0.8 eV
  • • 塑膠材料：Ea = 0.8-1.2 eV
  • • JEDEC 測試溫度：85°C, 125°C, 150°C

步驟 1：設計實驗

選擇至少 3 個溫度水平（例如：85°C, 100°C, 125°C），每個溫度測試 20-30 個樣品

步驟 2：收集失效數據

記錄每個樣品的失效時間 (TTF)

步驟 3：Arrhenius 繪圖

繪製 ln(MTTF) vs. 1/T，斜率 = -Ea/k

⚠️ 注意：

如果無實驗數據，可使用材料供應商提供的 Ea 值，或參考同類產品文獻

  • → NASA SSRI: Accelerated Life Testing
  • → JEDEC Standards (JESD22-A103)
  • → Arrhenius Equation (Wikipedia)
💧 濕度加速因子
Peck Model
公式：
AFRH = (RHalt / RHuse)n

參數 說明 典型值
RHuse 使用濕度 (%RH) 50-70%
RHalt 測試濕度 (%RH) 85-95%
n 濕度指數 2.0-3.0

  • • 一般電子產品：n = 2.0-2.5
  • • PCB 腐蝕：n = 2.5-3.0
  • • JEDEC HAST：130°C / 85% RH
  • • THB (溫濕偏壓)：85°C / 85% RH

步驟 1：固定溫度

選擇一個測試溫度（例如 85°C）

步驟 2：濕度變化

測試 3-4 個濕度水平（例如：60%, 75%, 85%, 95%）

步驟 3：參數估計

繪製 ln(MTTF) vs. ln(RH)，斜率 = -n

⚠️ 注意：

濕度與溫度常有交互作用，建議使用 Peck 完整模型（溫度+濕度）

  • → JESD22-A110 (HAST)
  • → JESD22-A101 (THB Test)
⚡ 電壓加速因子
IPL Model
公式：
AFV = (Valt / Vuse)β

參數 說明 典型值
Vuse 額定電壓 (V) 產品規格
Valt 測試電壓 (V) 1.1-1.5× Vrated
β 電壓指數 1.0-5.0

  • • 保守估計：β = 1.0
  • • 電介質擊穿：β = 2.4-2.7
  • • 電遷移：β = 1.5-3.0
  • • 測試電壓：不超過 1.5× Vrated

步驟 1：選擇電壓水平

通常選擇 3-4 個電壓水平（例如：1.0V, 1.1V, 1.2V, 1.3V）

步驟 2：固定溫度

在相同溫度下進行測試

步驟 3：IPL 繪圖

繪製 ln(MTTF) vs. ln(V)，斜率 = -β

⚠️ 注意：

過高電壓可能改變失效機制，建議不超過額定電壓的 150%

  • → Analog Devices: Voltage Acceleration
  • → Inverse Power Law (ReliaSoft)
🔄 熱循環加速因子
Coffin-Manson
公式：
AFTC = (ΔTalt/ΔTuse)β × (falt/fuse)α

參數 說明 典型值
ΔTuse 使用溫度範圍 (°C) 50-80°C
ΔTalt 測試溫度範圍 (°C) 100-165°C
fuse 使用頻率 (cycles/hr) 0.042 (1次/天)
falt 測試頻率 (cycles/hr) 1-3 cycles/hr
α 頻率指數 0.33
β 溫度指數 1.9 (焊點)

  • • 錫鉛焊點 (SnPb)：β = 1.9, α = 0.33
  • • 無鉛焊點 (SAC)：β = 2.65, α = 0.33
  • • 陶瓷材料：β = 5.0-6.0
  • • JEDEC條件：-55°C to +125°C (ΔT=180°C)
  • • IPC-9701：板級可靠度標準

步驟 1：定義循環剖面

確定最高/最低溫度和駐留時間（例如：-40°C/30min → +125°C/30min）

步驟 2：測試多個ΔT

使用 3 種不同的 ΔT（例如：100°C, 140°C, 180°C）

步驟 3：記錄失效循環數

定期檢查（例如：每100循環）直到失效

步驟 4：參數擬合

使用 Coffin-Manson 模型擬合 β 和 α

⚠️ 注意：

頻率效應：過快循環可能低估現場壽命（蠕變效應）

  • → JESD22-A104C (Thermal Cycling)
  • → IPC-9701 (Board-Level Reliability)
  • → Indium Corporation: TC Acceleration
📳 振動加速因子
IPL Model
公式：
AFVIB = (Galt / Guse)n

參數 說明 典型值
Guse 使用加速度 (g) 0.5-2.0 g
Galt 測試加速度 (g) 10-20 g
n 功率指數 6.0-12.0

  • • 焊點疲勞：n = 8.0-10.0
  • • PCB 裂紋：n = 6.0-8.0
  • • 一般桌上環境：Guse = 0.5-1.0 g
  • • 汽車環境：Guse = 2.0-5.0 g
  • • JEDEC測試：10-20 g (200-2000 Hz)

步驟 1：振動頻譜分析

使用加速度計測量實際使用環境的振動頻譜

步驟 2：設計振動測試

選擇 3-4 個加速度水平（例如：5g, 10g, 15g, 20g）

步驟 3：振動台測試

掃頻或隨機振動測試，記錄失效時間

步驟 4：參數估計

繪製 ln(MTTF) vs. ln(G)，斜率 = -n

⚠️ 注意：

振動測試需要專業設備（振動台），建議委託測試實驗室

  • → JESD22-B103B (Vibration Testing)
  • → MIL-STD-810 (Military Standard)
☀️ UV輻射加速因子
實驗比對
公式：
AFUV = tfield / taccelerated

參數 說明 典型值
tfield 現場暴露時間 (小時) 8760 (1年)
taccelerated 達到相同退化的測試時間 500-2000小時

  • • QUV測試：1000小時 ≈ 戶外 1-2年
  • • Xenon Arc：1000小時 ≈ 戶外 6-12個月
  • • 塗料/塑膠：AF = 5-15 (產品特定)
  • • 測試條件：340nm, 0.89 W/m², 60°C

步驟 1：定義失效標準

例如：光澤度降低50%、顏色變化ΔE≥5、拉伸強度下降20%

步驟 2：現場暴露

將樣品放置在實際戶外環境（如屋頂），定期檢測性能

步驟 3：加速測試

使用 QUV 或 Xenon 測試箱，控制：
• 光譜輸出（模擬太陽光）
• 溫度（60-70°C）
• 濕度（50-75% RH）
• 循環模式（UV/冷凝）

步驟 4：比對計算

當兩者達到相同失效標準時：
AF = tfield / taccelerated

⚠️ 重要限制：

• UV加速因子無法純理論計算，必須實驗確定
• 產品特定性：不同材料的AF差異很大
• 高度加速可能改變失效機制
• 建議查詢材料供應商數據或同類產品文獻

  • → ASTM G155 (Xenon Arc Exposure)
  • → ISO 4892 (Plastics Weathering)
  • → Atlas: Weathering Basics
  • → Accelerated Weathering Principles
🧪 化學濃度加速因子
IPL Model
公式：
AFCHEM = (Calt / Cuse)n

參數 說明 典型值
Cuse 使用濃度 (相對單位) 1.0 (基準)
Calt 測試濃度 (相對單位) 3-10倍
n 功率指數 1.5-3.0

  • • 金屬腐蝕：n = 2.0-2.5
  • • 聚合物降解：n = 1.5-2.0
  • • 鹽霧測試：5% NaCl溶液
  • • 相對濃度：以標準環境為基準 (1.0)

步驟 1：定義基準濃度

確定實際使用環境的化學物質濃度作為 Cuse = 1.0

步驟 2：濃度梯度測試

測試 3-4 個濃度水平（例如：1×, 3×, 5×, 10×）

步驟 3：固定溫度

在相同溫度下進行測試（通常結合 Arrhenius 模型）

步驟 4：參數估計

繪製 ln(MTTF) vs. ln(C)，斜率 = -n

步驟 5：濃度監測

使用 Beer-Lambert 定律監測溶液濃度：
A = ε × l × c

⚠️ 注意：

• 化學加速常與溫度有交互作用
• 過高濃度可能改變失效機制
• 需控制溶液 pH 值和氧氣濃度

  • → ASTM B117 (Salt Spray Test)
  • → Inverse Power Law (ReliaSoft)
☢️ 輻射劑量加速因子
TID Model
公式：
AFRAD = (Dalt / Duse)n
⚠️ 特殊應用：
僅適用於太空任務或核環境應用

參數 說明 典型值
Duse 使用環境累積劑量 10-150 krad
Dalt 測試累積劑量 100-300 krad
劑量率 測試劑量率 50-200 krad/hr
n 劑量敏感度指數 0.5-2.0
劑量敏感度指數 n 的意義：
  • n < 1：次線性，劑量率效應顯著（退火恢復）
  • n = 1：線性，標準 TID 模型
  • n > 1：超線性，高劑量協同效應

太空環境累積劑量 (TID)：
  • • LEO 低軌：5-15 krad/year
  • • GEO 同步軌：30-100 krad/year
  • • 極地軌道：10-20 krad/year
  • • 太陽風暴：可達數倍正常值
典型任務累積劑量：
  • • 5年 LEO 任務：50-75 krad
  • • 15年 GEO 任務：450-1500 krad
  • • 核設施儀表：視輻射源距離而定

步驟 1：確定任務劑量

根據軌道高度、任務時長、屏蔽厚度估算累積 TID
工具：SPENVIS (ESA), CREME96 (NASA)

步驟 2：選擇輻射源

• Co-60 γ 射線：能量 1.17-1.33 MeV
• 電子束：可調能量，適合 ELDRS 研究
• X 射線：低能量，模擬太陽耀斑

步驟 3：劑量率測試

至少測試 3 個劑量率（例如：10, 50, 300 rad/s）
確定是否存在 ELDRS (Enhanced Low Dose Rate Sensitivity)

步驟 4：劑量梯度測試

測試多個累積劑量點（例如：10, 30, 50, 100 krad）
繪製失效參數 vs. 累積劑量，擬合 n 值

步驟 5：退火效應

輻照後進行室溫或高溫退火（168小時）
評估參數恢復程度，確定長期效應

⚠️ 重要限制：

• 劑量率效應：低劑量率可能更危險（ELDRS）
• 器件相關性：不同製程差異極大
• 偏壓影響：必須在工作偏壓下測試
• 溫度交互：低溫可能抑制退火恢復

  • → MIL-STD-883 Method 1019 (TID Testing)
  • → ESCC Basic Specification No. 22900
  • → SPENVIS (Space Environment Info System)
  • → CREME96 (Cosmic Ray Effects)
⚙️ Eyring 模型 (進階)
應力交互
廣義 Eyring 公式：
t = A × (1/S)B × eEa/kT × eD×S/T
🎓 進階用途：
用於處理應力之間有交互作用的複雜情況

使用情境：
  • 溫度+電壓交互：高溫加劇電遷移效應
  • 溫度+濕度交互：溫度越高，濕度影響越顯著
  • HAST測試：高加速應力測試（T-RH-V同時作用）
何時不需要：
  • 應力相互獨立（大多數情況）
  • 單一應力測試
  • 消費性產品（簡化模型已足夠）

變數 說明
t 產品壽命
T 絕對溫度 (Kelvin)
S 非熱應力（電壓、濕度等）
Ea 活化能 (eV)
A, B, D 模型常數（需實驗確定）
D 交互作用項常數（關鍵）

步驟 1：階乘實驗設計 (DOE)

設計多水平多因子測試矩陣：
例如：3個溫度 × 3個電壓 = 9組實驗

步驟 2：每組測試

每個條件至少 20-30 個樣品

步驟 3：收集失效數據

記錄所有失效時間 (TTF)

步驟 4：參數擬合

使用專業軟件（ReliaSoft, Minitab）進行最大似然估計 (MLE)

步驟 5：模型驗證

在新的應力水平下測試，驗證預測準確性

⚠️ 複雜度警告：

• 需要大量實驗數據（通常 >200 樣品）
• 需要專業統計軟件
• 適用於高可靠度產品（軍用、航太）
• 大多數消費性產品使用簡化模型即可

  • → ReliaSoft: Eyring Relationship

📈 加速壽命測試 (ALT)
理論基礎
核心概念：
在高應力條件下測試產品，以在短時間內獲得長期壽命預測
✅ 本應用採用：
• 溫度：Arrhenius 模型
• 濕度：Peck's 模型
• 電壓：Inverse Power Law (IPL)
• 熱循環：Coffin-Manson 模型
• 振動：IPL 模型
• 進階：Eyring 模型（可選，考慮應力交互作用）

定義

加速壽命測試 (ALT) 是一種可靠度工程方法，通過將產品暴露於比正常使用條件更嚴苛的應力環境（高溫、高濕、高電壓等），在較短時間內誘發失效，從而推算產品在正常條件下的壽命。

為什麼需要 ALT？
  • 縮短測試時間：產品正常壽命可能達 10-20 年，無法等待實際驗證
  • 加速因子 (AF)：例如 AF=100，表示 1000 小時測試 = 11.4 年實際使用
  • 經濟效益：在產品設計階段即可識別弱點並改進
主要應力類型（本應用採用的模型）
  • 🌡️ 溫度 - Arrhenius 模型：
    AFT = exp[(Ea/k) × (1/Tuse - 1/Talt)]
  • 💧 濕度 - Peck's 模型：
    AFRH = (RHalt / RHuse)n
    結合溫度使用 (THB, HAST)
  • ⚡ 電壓 - Inverse Power Law (IPL)：
    AFV = (Valt / Vuse)β
    適用：電介質崩潰、電遷移
  • 🔄 熱循環 - Coffin-Manson：
    AFTC = (ΔTalt/ΔTuse)β × (falt/fuse)α
    適用：焊點疲勞、封裝應力
  • 📳 振動 - IPL 模型：

📊 Weibull 分析
統計分佈
CDF 公式：
F(t) = 1 - exp[-(t/η)β]
✅ 本應用支持多種方法（可選）：
中位秩方法：
• Benard's Approximation（預設推薦）
  F = (i - 0.3) / (N + 0.4) - 最常用，簡單準確
• 精確中位秩 (Exact Median Rank)
  F = Beta-1(0.5, i, n-i+1) - 使用 Beta 分佈，理論最準確
• 平均秩 (Mean Rank)
  F = i / (N + 1) - 最簡單，但偏差較大 回歸方法：
• RRY（預設推薦） - 最小化 y 方向殘差，n<15 適用
• RRX - 最小化 x 方向殘差
• MLE（最優） - 統計最優，n≥15 推薦

定義與應用

Weibull 分佈是可靠度工程中最廣泛使用的失效時間分佈，因其靈活性能模擬多種失效模式：從早期失效（嬰兒期死亡）到隨機失效，再到磨損失效。

兩個關鍵參數
  • 形狀參數 β (Beta)：控制失效率隨時間的演變行為
    • β < 1：失效率隨時間降低（嬰兒期死亡）
    • β = 1：失效率恆定（隨機失效，等同指數分佈）
    • β > 1：失效率隨時間上升（磨損失效）
  • 尺度參數 η (Eta)：特徵壽命，對應 63% 產品已失效的時間點
    • η 越大，失效發生越晚，產品壽命越長
    • η 只影響時間尺度，不改變失效率變化趨勢（由 β 決定）
實際意義範例

• 消費電子 (β=0.5)：製造缺陷導致早期失效，良品隨時間變穩定
• 軟體 Bug (β=1.0)：隨機觸發，失效率恆定
• 機械軸承 (β=3.0)：磨損累積，失效率隨時間快速上升

1. 中位秩回歸 (MRR)
  • 優點：簡單、視覺化、提供合理準確的 β 和 η 估計
  • 方法：在 Weibull 機率紙上繪製失效數據，擬合直線
  • 適用：小樣本 (<15) 或快速初步分析
2. 最大似然估計 (MLE)
  • 優點：統計最優，處理右設限數據能力強
  • 適用：較大數據集 (≥15)，需要嚴謹統計推論
  • 工具：需要專業軟件 (Weibull++, Minitab, JMP)
3. 線性回歸 (LR)
  • 方法：對 Weibull CDF 取對數兩次，轉換為線性關係
  • 適用：小數據集，教學演示
⚠️ 選擇建議：

對於工程應用，n < 15 使用 LR/MRR，n ≥ 15 使用 MLE。若有設限數據（部分樣品未失效），優先使用 MLE。

Weibull 圖解讀
  • 數據點落在直線上 → Weibull 分佈擬合良好
  • 下凹曲線 → 多重失效模式混合，需分段分析
  • 上凸曲線 → 可能有磨損後加速失效
β 值的工程意義
β 範圍 失效類型 管理對策
0.5-1.0 早期失效 加強 Burn-in、改善製造
~1.0 隨機失效 預防性維護無效，備用設計
2.0-4.0 磨損失效 預防性更換、設計改良
>5.0 快速磨損 緊急：設計缺陷，立即改進
高 β 值的迷思

高 β 值（如 β>5）不一定是壞事！若 η 足夠大，產品仍可非常可靠。關鍵在於綜合評估 β 和 η。例如：β=10, η=50年 的產品，在前 30 年幾乎不會失效。

🎯 零失效評估
Chi-Squared
MTBF 下限公式：
MTBF = 2T / χ²[(1-CL)/n, 2r+2]
🔍 特殊情境：
當測試未觀察到失效 (r=0) 時的可靠度評估方法
✅ 本應用採用：
• 統計方法：Chi-Squared 分佈
• 公式：MTBFlower = 2T / χ²(CL, df=2)
• 預設信心水準：60%（可調整至 90%/95%）
• 計算工具：scipy.stats.chi2.ppf
• 任務可靠度：R(t) = exp(-λt)（指數分佈假設）

問題描述

在高可靠度產品測試中，常因測試時間或樣本數限制，測試結束時未觀察到任何失效（r=0）。傳統 MTBF 估計公式 MTBF = T/r 無法使用（分母為零）。

為什麼不能說「MTBF = ∞」？

零失效並非代表產品永不失效，只是在有限測試時間內尚未觀察到失效。真實 MTBF 仍是有限值，但我們只能估計其下限。

Chi-Squared 分佈解法

利用卡方分佈 (χ²) 的統計性質，即使 r=0，仍可計算 MTBF 的信賴下限。這是一個保守估計，實際 MTBF 有 CL% 的機率大於此值。

⚠️ 關鍵假設：

零失效評估假設產品失效率恆定（指數分佈，β=1），這在「有用壽命期」通常成立。若產品處於磨損期（β>1），此方法會過於樂觀。

完整公式
MTBFlower = 2T / χ²[(1-CL)/n, 2r+2]
變數說明
T 總測試時間 = n × t (樣品數 × 測試時間)
r 觀察到的失效數（零失效時 r=0）
CL 信心水準 (常用 60% 或 90%)
χ² 卡方分佈臨界值，自由度 df = 2r+2
零失效時 (r=0)

當 r=0 時，自由度 df = 2，公式簡化為：
MTBFlower = 2T / χ²[(1-CL)/n, 2]

實例計算

• 總測試時間 T = 17,520 小時
• 零失效 (r=0)
• 信心水準 CL = 60%
• χ²0.4, 2 = 1.833
→ MTBFlower = 2×17,520 / 1.833 = 19,121 小時

結論：有 60% 的信心，真實 MTBF ≥ 19,121 小時。

常用信心水準
CL 應用場景 χ²值 (df=2)
60% 消費性產品、零失效測試標準 1.833
90% 工業產品、合約要求 0.211
95% 關鍵系統、醫療設備 0.103
99% 航太、軍用 0.0201
為什麼 60% 是零失效標準？
  • 平衡風險與成本：更高 CL 需要更長測試時間
  • 業界慣例：消費性電子產品普遍接受
  • 統計意義：當測試充分時，60% 已提供有用資訊
CL 對 MTBF 下限的影響

以 T=10,000 小時為例：
• CL=60% → MTBF ≥ 10,912 小時
• CL=90% → MTBF ≥ 94,787 小時
• CL=99% → MTBF ≥ 995,025 小時

CL 越高，MTBF 下限反而越大！這是因為要更有信心，必須假設更高的真實 MTBF。

何時使用零失效評估？
  • 高可靠度產品（MTBF > 100,000 小時）
  • 測試資源有限（時間/樣本數）
  • 需要快速評估可靠度下限
  • 驗證產品是否符合 MTBF 門檻要求
如何提高 MTBF 下限估計？
方法 1：延長測試時間

T 增加 2 倍 → MTBFlower 增加 2 倍（線性關係）

方法 2：增加樣本數

若無法延長單個樣品測試時間，增加平行測試樣品數同樣有效

方法 3：結合 ALT

在加速條件下進行零失效測試，再換算回使用條件（需注意失效機制一致性）

⚠️ 局限性：

• 僅提供下限，無法估計平均值或上限
• 假設失效率恆定（指數分佈），不適用磨損期
• 對測試時間敏感：T 太短會導致下限過低，無實用價值
